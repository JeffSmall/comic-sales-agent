## What it is
An **iOS app for tracking the market value of collectible comics** (graded CGC/CBCS and raw). The UI
is **generated by an AI agent and rendered as native, data-dense widgets** — not chat text. The user
**taps** to navigate/act and **types** (always-present input bar) when tapping isn't enough.
**Positioning:** *a tap-driven, AI-generated dashboard for your comic watchlist — see what your keys
are worth, by grade, at a glance.*

## User & mindset
A comic collector/investor who watches "key" issues and thinks in **grades, price ranges, recent
comps, and trends** ("what did a 9.8 New Mutants #98 last sell for; are 9.8s softening vs 9.4s?").
Quick glanceable check-ins; occasional deep dives and adds/edits.

## The one big idea
**Grade is the primary axis.** A 9.8 can be 5–10× a 9.4. Every sale carries its own grade, so the
product shows price **by grade** (medians, ranges, grade-tier matrices, "9.8s softening" trends).

## Core domain vocabulary
Comic (Title, Issue, Publisher, Notes) · Owned copy (Graded/Raw, Grader CGC/CBCS, Grade 0.5–10.0) ·
Sale/comp (price, date, grade-of-that-sale, edition newsstand/direct) · Price history (~90-day eBay
sold) · Derived: last, median, min–max range, % change — per book and per grade.

## Navigation (single surface, drill-in)
`Home = Watchlist  ⇄(tap a comic) Book Detail  →(tap "← Watchlist") back to Home`; `Home ⇄(tap ⚙)
Manage`. Empty watchlist → Welcome/first-run view. **Dashboard is tap-only** — footer = **⚙ gear
(Manage: add/remove)** + **"$" (Update Sales)**, no dashboard text field. "Back" is an in-content
control. Content **replaces in place** (drill-in, not a growing feed).

## Screens to wireframe
1. **Home / Watchlist (default landing, auto-loads, ≤12 books).** Vertical list of tappable rows; each
   row = title + issue + publisher (left), owned grade or "Raw" badge, **last sale price** (right,
   tabular), and a price ▲/▼ accent (+ optional inline sparkline). Sort chips (Value/Mover/Grade/
   Recent/Year). Footer: ⚙ Manage (left) · "$" Update Sales (right).
2. **Welcome / Empty (first run only).** Warm one-liner + "you're not tracking any comics yet" +
   guidance to add the first (Title, Issue, Grade, Graded/Raw) with an example. **Has a text field.**
3. **Book Detail — the dynamic market view** (absorbs trends). Top→bottom: "← Watchlist" back ·
   header (title+issue, publisher·date) · **FMV = median** hero + % change · MetricCards (Last,
   Median, Range, 30-day sparkline) · **30/60/90/ALL toggle + large trend line chart** ·
   **GradeTierMatrix** (grade × volume) · **Grade Variance** rows (grade·price·sparkline·HIGH/MED/LOW)
   · **Recent Transactions** comps table.
4. **Manage (⚙ gear).** Conversational add/remove comics (≤12) — the home for free-text entry.
5. **Update Sales ("$" footer icon).** Non-blocking background refresh + "updated 2h ago".
6. **States:** loading spinner; graceful sparse/empty grades; plain-language error card (never blank).
7. *(v2, optional)* price-alert push → deep-links into a Book Detail.

## Component vocabulary (build from these)
**WatchlistRow** (title/price/sparkline, no gridlines) · **Sparkline** (word-sized, accent dot) ·
**MetricCard** (number + label + optional delta, no borders) · **GradeTierMatrix** (dense grade×sales
grid) · **SmallMultiplesGrid** · **TextBlock** (sparse fallback). New looks = new/edited components,
not one-off art.

## Visual direction (Tufte data-ink)
Clean, dense, editorial/financial — like a well-set data table, not a flashy app. **Maximize
data-ink, no chartjunk** (no gridlines/borders/shadows). **Monochrome + one accent** reserved for
meaning (price up/down, selection, anomalies). **Direct labeling** (no distant legends).
**Right-aligned, decimal-aligned numerics — numbers are the hero.** Strong title hierarchy, quiet
metadata. **Design light + dark.** Subtle motion (drill-in, settle-to-top). App icon/identity open.

## Constraints
iOS phone portrait · catalog-constrained reusable components (not bespoke per screen) · dual input
(don't design away the input bar) · real, grade-keyed, sometimes-sparse data · keep views composable
and not huge (a current rendering size limit favors lean layouts; designing rich is fine — we're
lifting it).

## Open questions to resolve in design
1) Where do Update Sales / sort / filter live (toolbar vs chips vs menu)? 2) Guided add form vs pure
conversation? 3) Grade-tier visual at a glance (row sparkline) vs detail (full matrix)? 4) Accent
semantics — one accent for all, or an up/down pair? 5) Handling sparse/empty grades cleanly.
